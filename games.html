How it works
1. Before you bet, the server commits to a secret seed by showing you its SHA-256 hash.
2. Your result is HMAC-SHA256(serverSeed, clientSeed:nonce)
3. When you rotate seeds, the server reveals the real seed. The SHA-256 of it must match what was shown before — proving it was never changed.
4. You can then verify every single bet result using the verifier above.
